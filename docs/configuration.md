# Configuration

## What lives where

`.env` is generated by `./bootstrap.sh up`. Don't commit it — it contains
machine-specific Tailscale state. If you need to override a value, edit `.env`
and re-run `./bootstrap.sh up`; bootstrap reads the existing `.env` before
rewriting it, so manually set values (like `TAKEOVER_SUFFIXES`) survive the
rewrite.

`docker-compose.override.yml` is also generated by bootstrap (only when
Tailscale is present) and not committed. It adds Tailscale interface port
bindings to the Traefik service.

Everything below is what bootstrap writes or what the containers read.

## Variables

### `SUFFIX`

Default: `docker.localhost`

Read by: `discoverer`, `welcome`, written to `.env` by bootstrap.

The primary DNS suffix for all generated hostnames. Bootstrap writes this
once and doesn't change it on subsequent runs (it preserves the value from the
existing `.env` if present). Change it by editing `.env` directly, then
re-running `./bootstrap.sh up`.

`docker.localhost` resolves to loopback on every OS without any DNS config
(RFC 6761 reserves `.localhost`). Other useful values:

- `localhost` — shorter URLs, production-domain mirroring without the `orb.`
  prefix
- `local.dev` — requires DNS setup (dnsmasq or split-horizon DNS), not
  handled automatically

See [hostname-rules.md](hostname-rules.md) for the resulting URL shapes.

### `MACHINE_NAME`

Default: empty (auto-detected)

Read by: `discoverer`, `welcome`.

The short machine name used in tailnet hostnames. Bootstrap derives this from
`tailscale status --self --json` → `.Self.DNSName` (the DNS-safe slug
Tailscale assigns, not `HostName`, which can contain spaces). If Tailscale is
absent or returns no IP, this is left empty and tailnet hostnames are
disabled.

You can set this manually in `.env` if you want tailnet-style hostnames
without Tailscale, but you'll also need `TAILNET_DOMAIN` and you'll need to
handle your own DNS.

### `TAILNET_DOMAIN`

Default: empty (auto-detected)

Read by: `discoverer`, `welcome`.

The MagicDNS suffix for your tailnet, e.g. `tailXXXX.ts.net`. Bootstrap
derives this from `tailscale status --json` → `.CurrentTailnet.MagicDNSSuffix`.
Empty when Tailscale is absent.

### `TAILSCALE_IP`

Default: empty (auto-detected)

Read by: bootstrap (for the `docker-compose.override.yml` port bindings only).

The Tailscale interface IP (`tailscale ip -4`). Not passed into containers.
Used solely to bind Traefik on the right interface when Tailscale is present.

### `TAKEOVER_SUFFIXES`

Default: empty (takeover mode disabled)

Read by: `discoverer`, `bootstrap.sh` (to activate the `takeover` profile
and generate `dnsmasq.conf`).

Comma-separated list of additional DNS suffixes to serve. Each suffix must
match `^[a-z0-9][a-z0-9.-]*$` with no `..`, no trailing `.` or `-`.

Example:

```
TAKEOVER_SUFFIXES=orb.local
```

When non-empty, bootstrap:
- Generates `dnsmasq.conf` and starts the `dnsmasq` sidecar.
- Creates `/etc/resolver/<suffix>` files (with a confirmation prompt, or
  silently if `YES=1`).
- Adds `*.<suffix>` and `*.<project>.<suffix>` SANs to the wildcard cert.

Takeover mode is currently macOS-only (it requires `/etc/resolver`).

See [takeover-suffixes.md](takeover-suffixes.md) for full details.

### `TAKEOVER_DNS_PORT`

Default: `5300`

Read by: `bootstrap.sh` (written into `dnsmasq.conf` and `/etc/resolver`
files), `docker-compose.yml` (the dnsmasq host port binding).

The host port dnsmasq listens on. Change it if something else is already
using 5300.

### `TAKEOVER_ADDRESS_FLAGS`

Default: empty (synthesized by bootstrap)

Read by: `docker-compose.yml` (passed to dnsmasq as command-line flags in
earlier versions; now only used as a record in `.env`).

Bootstrap synthesizes this from `TAKEOVER_SUFFIXES` and writes it back to
`.env`. Operators don't need to set it manually; it's kept in `.env` for
visibility and debugging.

### `NETWORK_NAME`

Default: `dynomesh-net`

Read by: `discoverer` (via `NETWORK_NAME` env, falls back to
`inspect.DefaultNetworkName`).

The Docker network discoverer monitors for container membership. Change this
if you renamed the network. The network must be created (external) before
bootstrap starts the stack — bootstrap creates `dynomesh-net` by default.

### `FORCE_TAKEOVER`

Default: unset

Read by: `bootstrap.sh` (the conflict-detection gate).

Set to `1` to skip the conflict probe in `takeover_setup()`. Use this when
you know OrbStack isn't actually serving the requested suffix on this
machine but the probe is still triggering for some reason.

```bash
FORCE_TAKEOVER=1 ./bootstrap.sh up
```

See [takeover-suffixes.md](takeover-suffixes.md#conflict-detection-gate)
for when this matters.

### `YES`

Default: unset

Read by: `bootstrap.sh` (the resolver-file confirmation prompts).

Set to `1` to skip interactive prompts when creating or overwriting
`/etc/resolver/<suffix>` files. Useful for scripts and CI:

```bash
YES=1 ./bootstrap.sh up
```

### `DEBOUNCE_MS`

Default: `500`

Read by: `discoverer`.

Event debounce window in milliseconds. discoverer waits this long after the
last Docker event before triggering a reconcile. Increase it if your
environment produces noisy event bursts (e.g. a deployment that starts and
stops many containers in quick succession). Decrease it if you want faster
route propagation at the cost of more cert reissue calls (they're no-ops when
SANs haven't changed, so this is mainly a question of log noise).

## Auto-generated files (do not edit by hand)

- `.env` — rewritten on every `bootstrap.sh up`. Edit values here; they
  survive the rewrite.
- `docker-compose.override.yml` — Tailscale port bindings. Deleted and
  recreated on each run. If Tailscale is absent, the file is deleted.
- `traefik/dynamic/setup.yml` — setup-page routers. Deleted and recreated.
  If `MACHINE_NAME`/`TAILNET_DOMAIN` are empty, the file is deleted.
- `dnsmasq.conf` — dnsmasq address rules. Deleted and recreated when
  `TAKEOVER_SUFFIXES` changes.
- `traefik/dynamic/auto.yml` — live Traefik dynamic config, written by
  discoverer. Changes on every reconcile.
